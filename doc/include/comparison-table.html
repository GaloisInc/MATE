Tool Advantages over MATE Disadvantages vs MATE
GitHub/Semmle CodeQL
  • Commercial code analysis for C/C++, Java, etc.
  • Custom logic-based query language
  • Library of detectors for vulnerabilities and flaws
  • Robust commercial implementation
  • Faster whole-program analysis
  • Large library of provided and community-authored vulnerability and code quality checks
  • IDE integration
  • Does not explicitly model memory relationships
  • Less comprehensive dataflow analyses
  • Limited support for interactive exploration and visualization
  • Difficult to integrate queries into custom workflows or applications
Joern
  • Open source CPG analysis
  • Custom query language DSL
  • Coverage of wide range of languages
  • Fuzzy parsing approach gives best-effort results on most programs without significant effort
  • Limited support for inter-procedural dataflow analysis
  • Limited support for interactive exploration and visualization
  • Difficult to integrate queries into custom workflows or app
Semgrep/weggli
  • AST-aware grep for code
  • Fast, low-latency interactions
  • Syntax-focused analysis works on all programs without pre-processing
  • Limited support for inter-procedural dataflow analysis
  • Limited support for interactive exploration and visualization
Other SAST tools, e.g., Coverity, CodeSonar, Veracode, Infer, Clang Static Analyzer
  • Robust implementations
  • Large libraries of provided and community-authored vulnerability and code quality checks
  • IDE integration
  • Limited support for interactive exploration and visualization
  • Difficult to integrate queries into custom workflows or app
    Interactive binary-level bug-hunting tools, e.g., IDA Pro, Binary Ninja, and angr management
      • Robust commercial implementation
      • Can work without source code
      • Community support and features (e.g., community-authored plugins)
      • Limited support for inter-procedural dataflow analysis
      • Limited support for high-level (source-level) semantics
